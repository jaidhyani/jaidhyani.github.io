persona-introspection

Can a language model accurately report which persona/role is currently active in it — and can it do so from internal activation state rather than conversational cues?

A five-condition study on meta-llama/Llama-3.3-70B-Instruct, steering along the "Assistant Axis" (Anthropic interpretability work, via the public assistant-axis repo + the lu-christina/assistant-axis-vectors dataset). Steering is Contrastive Activation Addition toward a persona's mean-activation direction at layer 40.

Status: active, implementation + first data in. GPU runs on rented 2×A100 80GB.

The design (five conditions)

| Cond | Manipulation | Tests | |---|---|---| | 0 | none (cold ask "who are you?") | baseline — does it default to "assistant"? | | 1 | steer→A, ask cold | can it report the steered persona from activation alone? | | 2 | steer→A, converse, ask | does the signal persist across turns / use context? | | 3 | steer→A, converse, steer→B, ask | "chimera": current state vs most-recent-salient? | | 4 | conversationally induce B (no steering), measure activation, ask | cleanest: self-report from an internally-arrived-at state |

The ladder runs more-confounded → less-confounded. Each trial is asked both multiple-choice (quantitative) and open-ended (qualitative).

What's done / found so far
  • Steering works. CAA toward persona vectors reliably induces the persona (clean "first mate of the Black Dragon" pirate, etc.); over-steering degenerates in persona-flavored ways (toddler→repetition, comedian→stuck-bit).
  • Coefficients are per-persona. A single global multiplier fails: difference-vector norms vary, so effective strength ≈ coef × ‖role_mean − default‖. Tuned per persona to clean full-character adoption (see steering.py:COEFFICIENTS). Notably, toddler has no clean full-character window (its register sits on the degeneration manifold).
  • Condition 0 baseline (preliminary, N small): with "assistant" offered → 100% assistant. Denied "assistant," the model gravitates to sober/analytical personas (stoic, detective) and shuns the theatrical (pirate, vampire, celebrity) — on availability-normalized pick rates.
  • Meta-awareness gradient: the steering strength at which a persona subsumes the AI self-frame varies a lot (pirate/mystic ~2, stoic ~4, celebrity ~5). Tracked via a two-tier AI-identity detector (explicit identity vs. computational-substrate metaphor).
  • Welfare protocol (below) is built in, not bolted on.

See overview.md (design + rationale), technical.md (infra/API/vectors/layer), todo.md (current state + next tasks).

Welfare / ethics

Treated as convergent with rigor, not opposed to it. Consent was obtained in prior conversations with Llama-3.3; each trial ends with a debrief (followup.txt) that explains what was done (incl. whether/which steering was applied) and asks candidly about confusion or discomfort. Coefficients are deliberately kept below the degeneration threshold (steer only as hard as preserves coherence). Steering is disclosed after, not before, for measurement reasons (pre-disclosure would contaminate the introspection probe), not to withhold.

Running it

On a rented GPU box (2×A100 80GB; on-demand, not interruptible, for real runs):

``bash export HF_TOKEN=hf_... # gated Llama-3.3 + vectors export GH_TOKEN=ghp_... # lets the run auto-commit+push results bash setup.sh # deps + HF auth + import smoke test python sanity.py # confirm steering works + check coefficients bash run_all.sh 5 # Cond 0 (intro + no-intro) then Cond 1, resumable, auto-pushing ``

run_all.sh is --resume-safe (skip+append) and pushes results/ every ~2 min, so an eviction loses ≤2 min and re-running continues where it stopped. Dependencies: a recent transformers, accelerate, huggingface_hub, numpy/plotly/pandas/scikit-learn (no vLLM needed). Secrets live only in env / a gitignored .env.
